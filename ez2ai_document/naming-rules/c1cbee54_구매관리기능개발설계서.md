# 구매관리 기능 개발 설계서 (c1cbee54)

본 문서는 구매관리 프로그램 개발을 위한 기능 요구사항 및 설계 지침을 정리한 것입니다.

---

## 1. 프로젝트 개요
*   **목적**: 거래처, 품목, 구매 내역의 통합 관리 및 데이터 중복 입력 방지.
*   **핵심 가치**: **등록, 수정, 삭제, 조회** 기능의 최적화 및 **사용자 입력 편의성** 극대화.
*   **주요 흐름**: 거래처 등록 → 품목 등록 → 구매 내역 기록.

---

## 2. 기능별 상세 설계

### 2.1 거래처 관리
거래처 정보를 사전에 등록하여 구매 입력 시 목록에서 선택 가능하도록 구현합니다.
*   **필수 기능**:
    *   거래처 신규 등록, 수정, 삭제
    *   검색: `거래처명`, `사업자번호` 기준
    *   유효성 검사: **중복 등록 방지**
    *   구분: `매입처` / `매출처` 구분 관리

### 2.2 품목 관리
품목 정보를 사전에 등록하여 구매 시 단가 자동 호출 기능을 지원합니다.
*   **필수 기능**:
    *   품목 신규 등록, 수정, 삭제
    *   검색: `품목명`, `바코드` 기준
    *   유효성 검사: **중복 등록 방지**
    *   단가 관리: `매입 단가`, `매출 단가` 별도 관리

### 2.3 구매 관리
구매 내역을 입력하는 핵심 화면으로, 데이터 자동 연동 및 계산 기능을 포함합니다.
*   **필수 기능**:
    *   구매 내역 신규 등록, 수정, 삭제
    *   검색: `구매일자`, `거래처`, `품목` 기준
    *   자동화: 거래처 선택 시 `거래처명` 자동 표시, 품목 선택 시 `단가` 자동 표시
*   **금액 자동 계산 로직**:
    *   `공급가액` = `단가` × `수량`
    *   `부가세` = `공급가액` × 0.1
    *   `합계금액` = `공급가액` + `부가세`

## 3. 향후 확장 기능 (Roadmap)
추후 단계적으로 다음 기능들을 추가할 예정입니다.
1.  **통계**: 거래처별, 품목별 구매 집계
2.  **엑셀 연동**: 월별, 분기별 구매 현황 엑셀 다운로드
3.  **재고 관리**: 구매 내역 기반 재고 자동 반영

## 5. 개발 원칙 및 규칙
*   **데이터 무결성**: 거래처 및 품목 등록 시 **중복 방지 로직**을 반드시 적용합니다.
*   **사용자 편의성**: 구매 입력 시 거래처 및 품목 정보는 **목록 선택 방식**을 기본으로 합니다.
*   **자동화**: 구매 입력 화면에서 **금액 계산 로직**을 시스템이 자동으로 수행하도록 합니다.
