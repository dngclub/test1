# 구매관리 기능 개발 설계서 (c1cbee54)

본 문서는 구매관리 프로그램 개발을 위한 기능 요구사항 및 설계 지침을 정리한 것입니다.

---

## 1. 프로젝트 개요
*   **목적**: 거래처, 품목, 구매 내역의 통합 관리 및 데이터 중복 입력 방지.
*   **핵심 가치**: **등록, 수정, 삭제, 조회** 기능의 최적화 및 **사용자 입력 편의성** 극대화.
*   **주요 흐름**: 거래처 등록 → 품목 등록 → 구매 내역 기록.

---

## 2. 기능별 상세 설계

### 2.1 거래처 관리
거래처 정보를 사전에 등록하여 구매 입력 시 목록에서 선택 가능하도록 구현합니다.
*   **필수 기능**:
    *   거래처 신규 등록, 수정, 삭제
    *   검색: `거래처명`, `사업자번호` 기준
    *   유효성 검사: **중복 등록 방지**
    *   구분: `매입처` / `매출처` 구분 관리

### 2.2 품목 관리
품목 정보를 사전에 등록하여 구매 시 단가 자동 호출 기능을 지원합니다.
*   **필수 기능**:
    *   품목 신규 등록, 수정, 삭제
    *   검색: `품목명`, `바코드` 기준
    *   유효성 검사: **중복 등록 방지**
    *   단가 관리: `매입 단가`, `매출 단가` 별도 관리

### 2.3 구매 관리
구매 내역을 입력하는 핵심 화면으로, 데이터 자동 연동 및 계산 기능을 포함합니다.
*   **필수 기능**:
    *   구매 내역 신규 등록, 수정, 삭제
    *   검색: `구매일자`, `거래처`, `품목` 기준
    *   자동화: 거래처 선택 시 `거래처명` 자동 표시, 품목 선택 시 `단가` 자동 표시
*   **금액 자동 계산 로직**:
    *   `공급가액` = `단가` × `수량`
    *   `부가세` = `공급가액` × 0.1
    *   `합계금액` = `공급가액` + `부가세`

# 구매관리 시스템 개발 표준 가이드

## 1. 프로젝트 개요
본 프로젝트는 거래처, 품목, 구매 내역을 통합 관리하여 데이터 중복 입력 및 관리 효율성을 개선하기 위한 시스템 개발을 목적으로 합니다. **등록, 수정, 삭제, 조회**의 기본 기능에 충실하며, 사용자 편의성을 최우선으로 합니다.

## 2. 용어 정의 및 데이터 구조
시스템 내에서 사용되는 주요 엔티티 및 속성 정의입니다.

| 구분 | 주요 속성 | 비고 |
| :--- | :--- | :--- |
| **거래처** | 거래처명, 사업자번호, 구분(매입/매출) | 중복 등록 방지 필수 |
| **품목** | 품목명, 규격, 바코드, 매입단가, 매출단가 | 단가 자동 연동 |
| **구매 내역** | 구매일자, 거래처, 품목, 수량, 공급가액, 부가세, 합계금액 | 자동 계산 로직 적용 |

## 3. 네이밍 룰 및 식별자 규칙
*   모든 데이터는 고유 식별자를 통해 관리하며, **거래처명**과 **사업자번호**를 기준으로 거래처를 식별합니다.
*   품목은 **품목명**과 **바코드**를 기준으로 식별합니다.
*   시스템 내 모든 입력값은 중복 검증 로직을 거쳐야 합니다.

## 4. 코딩 및 비즈니스 로직 규칙

### 4.1. 구매 관리 자동 계산식
구매 내역 입력 시 다음의 산술 규칙을 반드시 준수합니다.
*   `공급가액` = `단가` × `수량`
*   `부가세` = `공급가액` × 0.1
*   `합계금액` = `공급가액` + `부가세`

### 4.2. 기능 구현 원칙
*   **거래처 관리**: 등록된 거래처는 구매 입력 시 목록에서 선택 가능해야 합니다.
*   **품목 관리**: 품목 선택 시 등록된 단가가 자동으로 호출되어야 합니다.
*   **구매 관리**: 거래처 선택 시 `거래처명`이, 품목 선택 시 `단가`가 자동으로 표시되어야 합니다.

## 5. 향후 확장 계획 (Roadmap)
본 시스템은 단계적 고도화를 고려하여 설계합니다.
1.  **데이터 집계**: 거래처별, 품목별 구매 현황 분석 기능
2.  **데이터 출력**: 월별, 분기별 구매 현황 엑셀 다운로드 기능
3.  **재고 연동**: 구매 내역 기반 재고 자동 반영 기능
