디자인 원칙
전 화면이 따르는 6 가지 핵심 원칙

1 · 토큰 우선 (Token-First)

색·폰트·간격·반경·그림자는 모두 --ez-* 토큰을 통해서만 사용합니다. 페이지 CSS 에서 #003087 같은 hex 직접 쓰기 금지. 다크 모드가 토큰 override 한 번으로 해결되는 이유이기도 합니다.

/* ❌ 잘못 */
.an-title { color: #1e3a8a; font-size: 14px; }

/* ✅ 올바름 */
.an-title { color: var(--ez-link); font-size: var(--ez-fs-base); }

2 · 단일 prefix per 페이지

한 페이지의 모든 클래스는 동일 2~3자 prefix 로 시작 (예: an- 공지, cc- 공통코드, in- 점검). 전역 CSS 충돌 차단 + 검색/디버깅 용이.

/* AnnouncementPage.css */
.an-page { ... }
.an-search { ... }
.an-title { ... }

단, 공통 컴포넌트가 제공하는 클래스는 그대로 사용합니다: eg-btn, eg-grid-*, eg-modal-*, eg-form-row, eg-toast-*, eg-pager 등.

3 · 컨트롤 36px / 본문 14px

버튼/입력/셀렉트는 모두 --ez-ctl-h (36px). 그리드 행은 --ez-row-h (40px), 헤더는 42px. 본문 글씨는 --ez-fs-base (14px).

왜? - 레거시 시스템 가 12px / 28px 였는데 노안 사용자가 많아 키웠습니다. 이 값들은 사용자 컨펌이 끝난 값이므로 임의로 줄이지 마세요.

4 · 행 클릭 → 수정 모달, + 등록 → 신규 모달

모든 목록 화면이 따르는 일관 패턴. ConfirmModal 로 확인 후 삭제, 토스트로 결과 알림. 모달 footer 의 삭제 버튼은 좌측, 취소/저장은 우측.

5 · 서버 한 번 + 클라이언트 페이지네이션

대부분의 목록은 검색 결과 전체(~수천 건)를 한 번에 가져와 클라이언트에서 페이징 (useMemo). 행 수 옵션은 20 / 30 / 50 / 100 / 500.

6 · 외부 UI 라이브러리 금지

⛔ AntD, MUI, shadcn, Bootstrap, Chakra 등 도입 절대 금지. 같은 룩앤필을 깨뜨립니다. 자체 컴포넌트 (Grid, DetailModal, ConfirmModal, useToast) 는 이미 충분히 모듈화돼 있으니 항상 재사용하세요. 차트만 예외 - Recharts 도입 시 사용자 승인 필수.
