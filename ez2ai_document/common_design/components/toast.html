표준 핸들러 패턴

1) 검색

try {
  const res = await api.get('/api/work-orders', { params: { ... }})
  setItems(res.data.items || [])
} catch (err) {
  if (err.response?.status === 401) { navigate('/login'); return }
  show('error', err.response?.data?.detail || '조회 실패')
}

2) 저장

try {
  await api.post('/api/work-orders', body)
  show('success', '저장되었습니다.')
  setForm(null); void fetchList()
} catch (err) {
  show('error', err.response?.data?.detail || '저장 실패')
}

3) 다건 처리

const res = await api.post('/api/work-orders/delete', { ids })
const ok = (res.data.deleted || []).length
const fail = (res.data.errors || []).length
show(fail ? 'info' : 'success', `${ok}건 삭제${fail ? ` · ${fail}건 실패` : ''}`)

가이드

  • 저장/삭제/등록 성공 → success
  • API 실패, 검증 실패 → error (백엔드 detail 문구 우선)
  • 안내성 정보 → info ("이미 처리된 항목입니다" 등)
  • 빈 결과는 그리드 emptyMessage 사용 - 토스트 X
  • 401 → /login 리다이렉트 + 토스트 X
